# Spec Driven Asteroids Skill

When activated, follow the standard SDD (Spec Driven Development) workflow.

## Guidelines
- **Requirements First**: Use EARS syntax.
- **Design Second**: Use Mermaid diagrams and DES- numbering.
- **Task Breakdown**: Create atomic tasks before coding.

## Available Tools
Use the connected MCP server to:
- `verify_ears_syntax(content)`
- `verify_mermaid_syntax(content)`
- `trace_requirements(tasks, requirements)`
